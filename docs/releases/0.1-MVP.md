Version 0.1 - MVP

Date: 2026-03-18

Tipo: MVP -> Preparación Beta (testing por parte de stakeholders)

Resumen:
Esta versión 0.1 corresponde al Producto Mínimo Viable (MVP) de Columna Pública. Su objetivo es entregar la funcionalidad básica para que los stakeholders evalúen el flujo principal: creación, edición y publicación de columnas, gestión de usuarios y revisión editorial.

Qué incluye:
- Interfaz de creación/edición de publicaciones (incluye carga y recorte de imagen 3:2).
- Flujo de envío a revisión y herramientas de aprobación/rechazo para editores.
- Persistencia de slug por publicación y enlaces amigables.
- Selector de etiquetas (Tags) y carga inicial desde .env.
- Registro de visitas y estadísticas básicas (/api/visits).
- Meta-tags dinámicos (OG/Twitter) para previews sociales.
- Persistencia de borradores en IndexedDB para autores.

Limitaciones conocidas:
- Índice único para slug no habilitado todavía (recomendado aplicar luego de migración controlada).
- Registro de visitas hace deduplicación básica; puede mejorar filtrado de bots.
- Pruebas E2E y automatizadas no añadidas aún.

Pasos para Beta:
1) Ejecutar migración para rellenar slugs existentes.
2) Habilitar índices únicos y resolver colisiones.
3) Preparar entorno de staging y coordinar sesiones de testing con stakeholders.
4) Añadir panel de métricas para visualizar visitas y comportamiento.
